Инструкция по работе с программой фильтрации

При запуске программы в командной строке необходимо подать файл, либо файлы, в формате .txt которые содержат в
случайном порядке строки, целые числа или вещественные числа. В качестве разделителя используется перевод строки.
Файлы читаются по очереди в соответствии с их перечислением в командной строке.
Программа фильтрует данные из полученных файлов и сохраняет целые числа в один выходной файл, вещественные в другой,
строки в третий. По умолчанию файлы с результатами располагаются в текущей папке с именами integers.txt,
floats.txt, strings.txt.
Дополнительно с помощью опции -o можно задать путь для результатов.
Опция -p задает префикс имен выходных файлов.
По умолчанию файлы результатов перезаписываются. С помощью опции -a можно задать режим добавления в существующие файлы.
По обработанным данным собирается статистика по каждому типу данных.
Вывести статистку можно в полном иди в коротком варианте, с помощью опции -f и -s соответственно.
Краткая статистика содержит только количество элементов записанных в исходящие файлы.
Полная статистика для чисел дополнительно содержит минимальное и максимальное значения, сумму и среднее.
Полная статистика для строк, помимо их количества, содержит также длину самой короткой строки и самой длинной.
Статистику по каждому типу данных выводится в консоль.

Инструкция по запуску программы

Для запуска програмы необходимо запустить командую строку и вызвать файл программы
Пример команды для запуска: "java -jar util.jar -s -a -p sample- in1.txt in2.txt"
Вызываем файл util.jar
Опции запуска: краткая статистика, дозапись в итоговые файлы, префикс для итоговых файлов "sample-", исходные файлы
in1.txt и in2.txt

Особенности реализации, не уточненные в задании:
1) В задании сказано "Полная статистика для строк, помимо их количества, содержит также размер самой короткой строки и
   самой длинной." Под словом "размер" можно понимать размер строки в байтах или количество символов в строке.
   В данной реализации считается количество символов в строке.
2) В задании сказано: "В процессе фильтрации данных необходимо собирать статистику по каждому типу данных".
   То есть статистика собирается в процессе чтения строк входящих файла. При дописывании существуюющих файлов,
   статистика собирается только по новым добавляемым в файл данным.
3) Допущена возможность создания более двух режимов вывода статистики (краткая, полная и другие на развитие),
   поэтому выбор режима вывода сделан через тип int вместо boolean.
4) В задании отсутствует информация - Нет ограничения на количество создаваемых новых директорий с помощью опции -o.
5) В задании отсутствует информация - При наличии пустой строки в исходных файлах, она пропускается.
6) В задании отсутствует информация - В программе не реализована обработка используемых символов (\,/,:,*,?,",<,>,|,+)
   в названии директории или файла будет выводиться информация о невозможности нахождения пути или файл будет
   сохраняться в исходной папке. Проверка осуществляется встроенными средствами системы Windows.
